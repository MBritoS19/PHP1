1. Mostrando dados atuais

Vamos criar um formulário que recebe o id de um produto e mostra os dados salvos em um formulário HTML: 

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Recebe id</title>
</head>
<body>
    <form action="dados.php">
	<label for="">ID que será atualizado:</label>
	<input type="text" name='id'><br>
	<input type="submit">
    </form>
</body>
</html>

O formulário envia os dados editados pelo usuário e o id (escondido) para a página que executa o comando UPDATE: 

<?php

// conexao
$servidor = 'localhost';
$banco = 'receitas';
$usuario = 'root';
$senha = '';

$conexao = new PDO("mysql:host=$servidor;dbname=$banco", $usuario, $senha);

echo "Conectado!<br>";

$codigoSQL = "UPDATE `ingredientes` SET `nome` = :nm, `quantidade` = :qtd WHERE `ingredientes`.`id` = :id";

try {
    $comando = $conexao->prepare($codigoSQL);

    $resultado = $comando->execute(array('nm' => $_GET['nome'], 'qtd' => $_GET['quantidade'], 'id' => $_GET['id']));

    if($resultado) {
	echo "Comando executado!";
    } else {
	echo "Erro ao executar o comando!";
    }
} catch (Exception $e) {
    echo "Erro $e";
}

$conexao = null;

?>

O comando UPDATE pode ser obtido editando uma linha da tabela no phpmyadmin.

2. O Problema da Injeção SQL

O que acontece se o usuário digitar '; DROP TABLE times; -- ao invés do valor de id?

Opção na preparação de comandos: 

<?php
$calories = 150;
$colour = 'red';
$sth = $dbh->prepare('SELECT name, colour, calories
    FROM fruit
    WHERE calories < ? AND colour = ?');
$sth->execute(array($calories, $colour));
?>

A linha $sth->execute(array($calories, $colour)); executa a consulta preparada, substituindo os marcadores de parâmetro pelos valores das variáveis correspondentes. Isso resulta em uma consulta SQL assim: 

SELECT name, colour, calories
FROM fruit
WHERE calories < 150 AND colour = 'red';

2.1. Exercício

  1. Ajuste todas as páginas mostra.php para usar um link com o id do time, produto ou livro que o usuário deseja atualizar ao invés de digitar o valor do id.
  2. Ajuste as páginas que recebem dados do usuário e executa comandos SQL para evitar injeções de código SQL malicioso.
